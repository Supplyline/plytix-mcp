## Bulk endpoint status (Step 0 evidence, 2026-06-10)

The shipped implementation (`docs/features/batch-update/REST-EVIDENCE.md`) concluded
`use_patch_loop`: no public async bulk endpoint is documented, so v1 batches over
documented single-product PATCH. Tonight's independent probes **reinforce** that
conclusion and add one datapoint (appended to REST-EVIDENCE.md):

- The official public Postman collection (88 endpoints, fetched from
  `apidocs.plytix.com/api/collections/42465620/2sBXijJrz3`) contains **zero**
  bulk/job/task endpoints.
- Read-only probes (GET / POST-search only, no write methods): `GET /api/v1/jobs/{id}` and
  `POST /api/v1/jobs/search` return **403 Forbidden** (route exists, permission-gated);
  every other guessed root (`/tasks`, `/api/v2/jobs`, `/bulk_actions`, `/processes`)
  returns 404. If Plytix ever exposes the async API, the job-status family is
  `/api/v1/jobs/...` — and **this account's API credential currently lacks that
  permission** (403 even on a nonexistent job id).
- Per the REST-EVIDENCE implementation rule, an undocumented permission-gated route is NOT
  `async_endpoint_confirmed`. The PATCH loop stands.

## Findings considered and rejected (do not re-audit)

- **CSRF on POST /authorize** (security agent): rejected — the form carries no ambient
  authority (no session/cookie); an attacker cannot submit the victim's credentials.
- **Non-timing-safe PKCE compare** as a standalone finding: comparing SHA256(verifier)
  against a challenge that already traveled in the URL leaks nothing useful; folded into
  plan 003 as one-line hygiene.
- **Plaintext creds in memory during /authorize** (SECURITY-06): by-design; encrypted
  before storage; the agent itself concluded no fix needed.
- **`products_get_full` payload bloat** (PERF-02): by design — "full" is the tool's
  contract; agents needing less use `products_get`/`products_search`.
- **README tool-count drift** (DOCS-01): rejected — recount confirms 47 stdio / 44 worker
  exactly as README states.
- **Tool-surface drift between stdio and worker** (part of DEBT-01): the *drift* claim was
  false — surfaces are in exact parity (47 vs 44 + 3 intentionally local-only). The
  *duplication* is real → plan 004.
- **CI integration tests use dummy creds** (TESTS-02): by design (no secrets in CI); they
  are MCP-protocol smoke tests. Optional gated live-API job noted in plan 002 maintenance.
- **zod v4 migration**: rejected for now — MCP SDK peer-deps zod 3; no concrete cost to
  staying.
- **Extensions/worker docs gaps** (DIRECTION-02/03): real but minor; revisit after 004.
